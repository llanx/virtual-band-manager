# Song DNA Registry

> A living reference updated after every finalized song. Consult during Phase 1 (Concept) and Phase 3 (Lyrics) to avoid repetition and ensure each track has a distinct identity.

## How to Use This File

- **During writing (Phases 1–4):** Do NOT consult this file. Write freely without pattern constraints.
- **Phase 4.5 (Optional rewrite aid):** If the user is unhappy with the Phase 4 output and wants to check for unintentional repetition across the catalog, consult this file. Flag overlapping `[SATURATED]` or `[WATCH]` patterns and suggest alternatives. The user decides which to revise.
- **Phase 5 (Final Output):** After the song is finalized and approved, update this file — add a new fingerprint row, scan lyrics for all tracked categories, update counts and status flags.

### Status Thresholds

| Uses | Status | Rule |
|------|--------|------|
| 1 | `OK` | Free to use |
| 2 | `[WATCH]` | Use at most once more, vary execution |
| 3+ | `[SATURATED]` | Do not reuse unless user explicitly approves |

> When the catalog reaches 8+ songs, raise thresholds: `[WATCH]` = 3+, `[SATURATED]` = 5+.
